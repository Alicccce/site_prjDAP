Сведения (автоматизированная система)
Наименование АС: «Персонализированный навигатор карьеры и обучения».
Цели создания АС: Сокращение времени пользователя на анализ требований рынка труда. Повышение релевантности рекомендаций по профессиональному развитию.
Назначение АС:Персонализированная навигация в плане обучения. Система предназначена для сбора и анализа данных о вакансиях, для сопоставления их с профилем пользователя, формирования персонализированных планов обучения.

Начало: 06.02.2026 
Окончание: 01.09.2026
Объектом автоматизации является деятельность пользователей по анализу рынка труда и планированию собственного профессионального развития.
Условия эксплуатации:Система функционирует в круглосуточном режиме, доступ к ней осуществляется пользователями через сеть Интернет с использованием стандартных веб-браузеров. Основные пользователи, разбитые на два сегмента, описаны в разделе ЦА данного дневника проекта.
Функциональные требования
1. Требования к структуре и функционированию системы
Подсистема сбора данных (ПС СД):Выполняет парсинг данных с HH.ru и других ресурсов.
Подсистема аналитики и AI-агента (ПС АА):Модуль на базе AI для анализа текстов вакансий, выявления скилов, сопоставления их с профилем пользователя, генерации планов обучения.
Подсистема взаимодействия с пользователем (ПС ВП):Веб-интерфейс (фронтенд) и API (бэкенд).
Подсистема хранения данных (ПС ХД):База данных для хранения информации о вакансиях, пользователях, их прогрессе и сгенерированных планах.

3. Требования к информационному обеспечению.
3.1 Состав данных: вакансии (название, требования), пользователи (email, профиль навыков), планы обучения. Система должна обеспечивать целостность и непротиворечивость данных на уровне БД. 
3.2 Интерфейс системы и вся выходная документация должны быть на русском языке. Допускается использование англоязычных терминов.
Нефункциональные требования  (по ИСO 25010)
1. Производительность
Анализ вакансии по ссылке — не более 10 секунд.

Генерация плана обучения — не более 30 секунд.

Система выдерживает до 20 одновременных пользователей.

Время загрузки страниц — до 4 секунд.

2. Надёжность
Коэффициент готовности системы: не менее 0.95. Время восстановления после сбоя (RTO): не более 4 часов. Обеспечение защиты от сбоев парсера (использование прокси, ротация User-Agent) - работа при блокировках. При недоступности источника данных (HH.ru) система использует последнюю успешно загруженную копию данных.
Автоматическое восстановление после сбоев — не более 30 минут.
Организацией бесперебойного питания технических средств.
3. Безопасность
Обработка персональных данных пользователей (ФИО, email, навыки) должна осуществляться в соответствии с Федеральным законом от 27.07.2006 № 152-ФЗ «О персональных данных». Все внешние ссылки в планах обучения должны проходить проверку на принадлежность к "белым" спискам ресурсов для исключения фишинга, нарушения авторских прав или вредоносного кода.
Обязательное использование HTTPS.
4. Требования к эргономике: Интерфейс должен быть интуитивно понятным, не требующим специального обучения для пользователей возрастной группы 20-45 лет. Формы ввода данных должны быть максимально простыми для восприятия. Информация должна иметь корректное отображение во всех браузерах (Chrome, Firefox, Yandex) и адаптируется под мобильные устройства.
